OpenSSL Interoperability

Intro & Setup Test Lockify's Encryption Test Lockify's Decryption

To help users verify our encryption, we've made Lockify interoperable with OpenSSL 1.0+ — a leading, open source cryptography library. So that you can verify this for yourself, we've built this step-by-step test tool that you can use in conjunction with your computer's Command Line Interface (CLI).

Note that like all Lockify encryption and decryption, this tool runs completely in your browser. If you like, you can even turn off your Internet connection.

Need to install/update to OpenSSL 1.0+?
Need to install/update to OpenSSL 1.0+?

Lockify is interoperable with OpenSSL 1.0+, as earlier versions don't support HMAC.

Unfortunately and despite having been released early in 2010, OpenSSL 1.0+ is still not widely deployed.

  • Depending on their package manager and its repositories, *nix users may or may not have Open SSL 1.0+.
  • Mac OS X has not yet upgraded to OpenSSL 1.0+.

If you need to update OpenSSL , we recommend doing it in a virtual machine, such as those that run in VMWare or VirtualBox, so as not to disrupt any of your OpenSSL-dependent system software.

Instructions for Windows Users

You must have Cygwin installed to use OpenSSL.

  1. Download/upgrade it from Cygwin .
  2. During installation, select the following packages to install:
    • OpenSSL Base Environment located under the category "Net"
    • VIM Enhanced VI Editor located under the category "Editors"
  3. Add Cygwin to your system path with this command:
    path = %PATH%;C:\cygwin\bin\;
